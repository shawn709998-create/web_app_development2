# 產品需求文件 (PRD)：個人記帳簿系統

## 1. 專案概述
### 背景與動機
在現代忙碌的生活中，許多人常常不清楚自己的金錢流向，導致月底成為「月光族」或無法有效達成儲蓄目標。市面上的記帳軟體有時過於複雜，我們需要一個簡單直覺的個人記帳系統，讓使用者能快速記錄收入與支出，並自動統計餘額與財務狀況，幫助使用者輕鬆培養記帳與理財習慣。

### 目標用戶
一般上班族、學生、自由工作者，以及想培養記帳與理財習慣的個人使用者。

### 核心價值主張
提供簡單、快速、直覺的記帳體驗，結合自動化的統計與圖表分析，讓使用者能一目了然地掌握個人財務狀況，進而達成財務目標。

## 2. 功能需求

### 1. 記錄收入與支出
- **功能描述**：使用者可以新增、編輯、刪除每筆收入或支出紀錄，包含金額、日期、分類與備註。
- **使用者故事**：作為一名使用者，我希望能夠快速輸入我的每日花費與收入，以便準確記錄我的財務流向。

### 2. 顯示餘額統計
- **功能描述**：系統首頁或儀表板自動計算並顯示特定期間的總收入、總支出與目前剩餘金額。
- **使用者故事**：作為一名使用者，我希望能一眼看出當月的總收支與結餘，以便隨時掌握目前的財務狀況。

### 3. 分類管理
- **功能描述**：系統提供內建的收支分類（如飲食、交通、薪水等），並允許使用者新增、編輯或刪除自訂分類。
- **使用者故事**：作為一名使用者，我希望能夠自訂收支分類，以便更符合我個人的消費習慣。

### 4. 圖表分析
- **功能描述**：以圓餅圖呈現支出比例，以折線圖或長條圖呈現收支趨勢。
- **使用者故事**：作為一名使用者，我希望透過圖表查看支出比例與趨勢，以便找出可以節省開銷的項目。

### 5. 查詢與篩選
- **功能描述**：提供依據日期區間、分類或關鍵字進行搜尋與篩選歷史紀錄的功能。
- **使用者故事**：作為一名使用者，我希望可以搜尋特定日期或分類的帳目，以便對帳或回顧過去的花費。

### 6. 預算設定
- **功能描述**：使用者可設定每月總預算或各分類預算，當支出接近或超過預算時系統提供超支提醒。
- **使用者故事**：作為一名使用者，我希望能夠設定每月預算並在即將超支時收到提醒，以便控制花費避免透支。

## 3. 非功能需求
### 技術限制
- 前端：使用 HTML / CSS / JavaScript 搭配 Jinja2 模板引擎進行頁面渲染。
- 後端：使用 Python 搭配 Flask 框架。
- 資料庫：使用 SQLite 作為關聯式資料庫。

### 效能與安全考量
- **效能**：頁面載入速度需快速，圖表渲染不可造成瀏覽器卡頓，查詢功能在資料量增長時仍需保持合理的回應時間。
- **安全性**：防止 SQL Injection 與 XSS 攻擊，確保使用者輸入資料的驗證與清洗。確保敏感資料的處理（若未來加入身分驗證需加密密碼）。

## 4. MVP 範圍
### Must Have (必須具備)
- 記錄收入與支出（包含新增、檢視、編輯、刪除）。
- 顯示餘額統計（總收、總支、剩餘金額）。
- 基本分類管理（內建分類，可不含自訂分類）。

### Should Have (應該具備)
- 自訂分類管理（新增、編輯、刪除）。
- 查詢與篩選（依日期與分類篩選）。
- 基礎圖表分析（如當月支出比例圓餅圖）。

### Nice to Have (有會更好)
- 進階圖表分析（收支趨勢折線圖）。
- 預算設定與超支提醒。
- 匯出報表功能。
